SAN & Storage Device Management

Service summary

The SAN & storage device management service is a Cegeka managed service which consist of the monitoring, management, maintenance, supervision, administration and planning of all functions for Storage systems.

Standard service components

The following standard features are included in this Service:

General activities

  • Monitoring and controlling storage performance according to data management policies[1]
  • Monitoring storage constantly and escalating data growth when it reaches a critical point to ensure timely procurement of sufficient storage.
  • Proposing a data management plan to manage data growth and capacity
  • Reports on performance and capacity

Daily operations and administration

  • Storage Provisioning & Management
    • Defines physical and logical layout (arrays, groups, volumes, pools, LUNs)
    • Creates new LUNs
    • Removes LUNs
    • Creates new Shares
    • Removes Shares
    • Manages host mappings (hosts, host groups, host types)
    • Backs up Device configuration
    • Masks Devices to HBA
    • Manages unused and used resources (disks, ports, controllers)
    • Determines failed hardware components (disk drives, power supplies, fans, controllers, ports)
    • Coordinates array capacity decision making
    • Coordinates disk types decision making
    • Troubleshoots Hardware issues
    • Logs Hardware issues with vendors
    • Manages Volume size (grow/shrink)
    • Manages volume location/relocation/migration within the same array or between arrays
    • Migrates RAID levels
    • Reports on number of disks available and used[2];
    • Remote Updates Microcode / firmware (our recommendation is that all these activities are done onsite)”
    • Documents Trend reporting on resource usage
    • Performs Storage space validations (trend analysis)
    • Configures replication
    • Manages deduplication and compression
    • Manages disk alignment
  • Storage Performance Management
    • Performs Configuration Monitoring
    • Performs Logs & Alert monitoring
    • Manages Performance Monitoring and Tuning
    • Manages Performance Analysis (trend analysis)
  • Storage maintenance
    • Monitors and Troubleshoots Proactively
    • Follows up and interacts with Vendor
    • Implements and coordinates Array Vendor Best Practices
    • Designs Scalability
  • Fabric Switch Management
    • Configures Domain Parameters
    • Configures interfaces on SAN Switch
    • Configures fiber channel aliases for new host
    • Configures & Managing VSANs
    • Configures high availability on controller level
    • Configures Inter-Switch links
    • Configures Port-channels / Trucking
    • Configures & Manages Zones
    • Configures Fiber channel routing services & protocols
    • Configures users & common rules
    • Configures System message logging
    • Configures IP Storage
    • Configures FCIP & iSCSI
    • Performs Problem Determination and Resolution
  • Fabric switch monitoring
    • Proactively Monitors and Troubleshoots
    • Configures SNMP alerts & monitoring
    • Troubleshoots & diagnoses Fabric & Switch Issues (Hardware & Software)
    • Performs Impact & Performance analysis
  • Storage Volume & File System Provisioning
    • Allocates / De-allocates Volumes/Logical disks
    • Manages Nomenclature for Volumes
    • Performs Volume Management (Grow/Shrink)
    • Provisions & designs new disks
    • Performs LUN Management/ Exports Management
    • Adds and Removes Disks
    • Performs Cluster Configuration & Management
    • Monitors and resolves issues with the synchronization to a secondary SAN
    • Stops Synchronization as requested
    • Reinstates Synchronization as requested
  • Storage Snapshot Management
    • Performs Replication and Recovery
    • Performs Snapshot Management
    • Coordinates Hardware Maintenance & Support
