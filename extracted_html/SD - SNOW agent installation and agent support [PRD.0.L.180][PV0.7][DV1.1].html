SNOW Installation and agent support

Service Model

Software Management Service.

Cegeka installs and support the client software running on Cegeka managed servers for the purpose of the Customer.

Service summary

Software Management Services

The Service provides the tasks and activities needed to install and support the SNOW (SNOW Software Asset Management) agent on servers (The Application) from a technical perspective. Cegeka unburdens the customer with all technical aspects of the agent installation and support.

The Standard Service features are focussed to install the agent and support on the agent in case of malfunction.

Maintenance and documentation of the environment is part of the service.

Snow Database (instances) and backend servers are managed by Cegeka as part of SD OS management and database management. SNOW team will support these teams.

Advise and support for using the user interface (Snow portal)

Advise and support installation new features like Snow Atlas.

The ownership and management of the SNOW Asset Management is in hands of the Customer. Cegeka only installs and supports the agents running on the servers managed by Cegeka.

The Customer is responsible for the functional SNOW usage and for checking the agents reporting to the central SNOW application.

The Application

SNOW is a Software Asset Management tool based on agents running on servers and workstations.

The agents scan the local host and report back to a central SNOW entity where consolidation is done.

The Application in this context is only the SNOW agent running on the Cegeka managed servers.

Standard Service features

Application knowledge

Cegeka ensures skilled engineers to install and support the SNOW agents according to customer requirements.

Technical setup

Cegeka performs the installation and support based on the customer requirements. General technical setup is documented by Cegeka and shared with the customer. Documentation will be updated in case set up will change.

Technical management

Technical management

There is no technical management included in the service, apart from the installation of the agent conform the procedure delivered by the customer. The initial setup is performed by SNOW and Cegeka will support and assist SNOW with the installation.

Please note that the OS management of the server running the Snow application is managed by Cegeka (based on the advice from SNOW).

Patching

Installation of new versions (updates of SNOW agents) will be done by Cegeka in collaboration with Customer license team. Triggers for updates of agents can be triggered based on a feature request (Cegeka or Customer) or on request of Snow.

Life Cycle Management

LCM is included in the service.

Monitoring

Monitoring of the Snow Agents installations is done based on health-status report.

Requests

Both the Customer and Cegeka can issue a Request for Change (RFC) or a Request for Information (RFI).

There are multiple types of requests:

  • Service Request
    • This is a pre-defined request
    • Some can be requested via Horizon: these are pre-approved and pre-defined requests
    • Service request that is not pre-approved will be executed after approval by the CAB
    • Service Requests are executed as a change ticket and will be delivered within the indicated lead time.
    • The pre-defined execution is optimal automated.
    • This is used for requests that are often needed (multiple times per month).
    • These have a fixed price.
  • Normal Changes :
    • these are other non pre-defined requests
    • they need an evaluation and analysis where Cegeka needs to deliver a proposal and impact analysis to address the change request (with a T&M estimation).
    • A successive review, approval and planning (maintenance window) by a CAB of the proposal is needed prior to execution.
    • These are billed on T&M basis.

The handling of Requests is subject to the regular change management process (see Cross-functionals).

RACI role definition:

  • R-Responsible: The party who is responsible for the execution of the activity.
  • A- Accountable: The party ultimately accountable for the correct and thorough completion of the task and/or activity as performed by the Responsible. For each task and/or activity exactly one Accountable must be specified. The Accountable has the mandate to accept/dismiss on work provided by the Responsible. The Accountable is by default reactive.
  • C-Consulted: The party who must be consulted and must approve or provide input prior to execution. This party (partly) gives direction to the result. This is about two-way communication.
  • I-Informed: That is the person/party who is informed about the decisions, progress and results achieved, so that the next step can be taken. This is about one-way communication.
